Movecost QGIS Plugin

R-based tool for calculation of slope-dependent accumulated cost surface, least-cost paths, and least-cost corridors related to human movement across the landscape.

Requirements

  • QGIS 3.0 or higher (compatible with QGIS 4)
  • Processing R Provider plugin
  • R with packages: movecost, sf, terra, raster

Available Functions

movecost

Calculates accumulated anisotropic slope-dependent cost of movement across the terrain and least-cost paths from a point origin to one or more destinations.

movebound

Calculates slope-dependent walking cost boundaries (isolines) around one or more point locations.

movecorr

Calculates the least-cost corridor between two point locations.

movealloc

Calculates slope-dependent cost allocation from multiple origin points, assigning each cell to its nearest origin by walking cost.

movecomp

Compares least-cost paths generated using different cost functions to analyze how different movement models affect path selection.

movenetw

Calculates a least-cost path network connecting multiple point locations.

moverank

Ranks destination points by their walking cost from an origin point.

Common Parameters

DTM/Area_of_interest: Digital Terrain Model raster or polygon to download DTM from online sources.

Origin: Point location(s) from which costs are calculated.

Destination: Point location(s) to which least-cost paths are calculated.

Move: Cell connectivity (4, 8, or 16 directions).

Function: Cost function to use (see table above).

Time: Time unit ('h' for hours, 'm' for minutes).

Cognitive_Slope: Use perceptual slope instead of real slope.

Walker_Body_Weight: Walker's weight in kg (default: 70).

Carried_Load_Weight: Load weight in kg (default: 0).
